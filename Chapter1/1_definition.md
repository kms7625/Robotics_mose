# 1. 문제 정의

## 1.1 배경

제조 현장에서 부품과 준비된 제품을 공정 간에 운송하는 작업은 현재 수작업으로 이루어지고 있다. 이 과정은 반복적이고 단순하지만 인력이 지속적으로 투입되어야 하며, 운송 경로와 시간에 대한 체계적인 기록도 부재한 상황이다.

## 1.2 핵심 문제

공정 간 부품·제품 운송을 자동화하기 위한 프로토타입 운반 로봇이 필요하다. 이 로봇은 하얀색 바닥에 검은색으로 표시된 운송 경로를 스스로 인식하고 따라 이동하며, 추후 평가·검토를 위해 실제 이동 기록 데이터를 수집·저장해야 한다.

# 2. 요구 사항

## 2.1 필수 요구 사항

| 번호 | 항목           | 세부 내용                                                                           |
| ---- | -------------- | ----------------------------------------------------------------------------------- |
| R1   | 경로 인식      | 하얀색 바닥 위 검은색 선(폭 약 2~3cm)을 광/적외선 센서로 인식한다                   |
| R2   | 자율 주행      | 인식된 선을 따라 자율적으로 이동하며, 직선·곡선 경로 모두 추종 가능해야 한다       |
| R3   | 경로 이탈 복귀 | 경로에서 벗어났을 경우 자동으로 복귀 동작을 수행한다                                |
| R4   | 데이터 수집    | 이동 시각(타임스탬프), 센서 상태, 구간 정보를 1초에 1회 이상 기록한다               |
| R5   | 데이터 저장    | 수집된 데이터는 CSV 또는 텍스트 로그 형식으로 저장된다                              |
| R6   | 데이터 열람    | 운행 종료 후 저장된 데이터를 외부 장치(PC 등)에서 확인할 수 있어야 한다             |
| R7   | 전원           | 배터리로 구동되며 1회 충전으로 최소 30분 이상 운용 가능해야 한다                    |
| R8   | 적재 용량      | 최소 1kg의 부품·제품을 안전하게 적재하고 운반할 수 있어야 한다                     |
| R9   | 장애물 대응    | 경로 위 장애물 감지 시 정지하고, 장애물 제거 후 자동 재출발하거나 경보를 발생시킨다 |
| R10  | 데이터 전송    | 운행 종료 후 USB 케이블 연결을 통해 PC로 로그 파일을 전송할 수 있어야 한다          |

## 2.2 추가 요구 사항 (선택)

- 운행 시작·종료를 물리적 버튼 또는 신호로 제어할 수 있으면 좋다
- 경로상의 특정 지점(예: 정차 구역)에서 일시 정지 후 재출발하는 기능이 있으면 좋다

# 3. 합리적 가정

과제에 명시되지 않은 사항에 대해 아래와 같이 합리적으로 가정한다.

**[환경]**

- 주행 환경은 실내 평탄한 바닥이다
- 조명은 일반 실내 형광등 환경을 기준으로 한다
- 경로 선의 폭은 약 2~3cm이다
- 경로에는 직선과 완만한 곡선이 포함되며, 급격한 꺾임은 없다

**[로봇 사양]**

- 최대 주행 속도는 안전을 위해 0.5m/s 이하로 제한한다
- 로봇 크기는 경로 폭의 3~5배 이내 소형으로 제작한다
- 소형 마이크로컨트롤러(Arduino 또는 Raspberry Pi 계열) 기반으로 구현한다
- 적재 하중 최대 1kg을 가정하며, 적재함은 로봇 상단에 고정형으로 설치한다
- 장애물 감지용 초음파 또는 IR 거리 센서를 전방에 1개 이상 배치한다진

**[데이터]**

- 데이터 저장 매체는 SD카드 또는 내장 플래시 메모리를 사용한다
- 전원이 꺼지더라도 저장된 데이터는 유지되어야 한다
- 데이터 전송 방식은 USB 케이블을 통한 파일 복사를 기본으로 하며, 무선 전송은 추후 확장 기능으로 고려한다

**[프로토타입 범위]**

- 이번 개발은 실제 공장 도입용이 아닌 기능 검증 목적의 프로토타입이다
- 완성도보다 핵심 기능(경로 인식, 자율 주행, 데이터 수집)의 동작 여부가 평가 기준이다

# 4. 성공 기준

다음 조건을 모두 만족하면 정의한 문제가 해결된 것으로 판단한다.

1. 로봇이 검은색 경로를 인식하고 이탈 없이 끝까지 주행한다
2. 주행 중 이동 기록 데이터가 실시간으로 수집·저장된다
3. 운행 종료 후 저장된 데이터를 USB를 통해 외부 PC에서 확인할 수 있다
4. 1kg 이하의 적재물을 싣고 정상 주행이 가능하다
5. 경로 위 장애물 감지 시 로봇이 정지하고 적절한 대응 동작을 수행한다
