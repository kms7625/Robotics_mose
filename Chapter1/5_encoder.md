# 엔코더(Encoder) 조사 보고서

---

## 1. 엔코더 모터의 정의

**엔코더 모터**란 DC 모터에 **엔코더(위치/속도 측정 장치)** 가 부착된 모터이다.
모터가 얼마나 회전했는지를 펄스 신호로 출력하여, 마이크로컨트롤러가 속도와 거리를 정확하게 계산할 수 있게 한다.

```
  [DC 모터] ──── [감속기어] ──── [엔코더 디스크]
                                       │
                                  펄스 신호 출력
                                       │
                              [마이크로컨트롤러]
```

## 2. 엔코더부의 역할과 구성 방식

### 2.1 역할

| 역할        | 내용                                        |
| ----------- | ------------------------------------------- |
| 회전 감지   | 바퀴/모터가 얼마나 회전했는지 측정          |
| 속도 계산   | 단위 시간당 펄스 수로 RPM 계산              |
| 거리 계산   | 총 펄스 수로 이동 거리 계산                 |
| 피드백 제어 | 목표 속도와 실제 속도 비교 → PID 제어 입력 |

### 2.2 구성 방식

#### 광학식 엔코더 (Optical Encoder)

- 슬롯이 뚫린 **인코더 디스크** + **적외선 LED** + **포토트랜지스터**
- 디스크가 회전하면 빛이 슬롯을 통과할 때마다 펄스 발생
- 정밀하고 내구성 높음

```
     ┌──────────┐
     │  ● ● ●  │  ← 슬롯이 뚫린 디스크
     │ ●     ● │
     │  ● ● ●  │
     └──────────┘
     IR LED ↑  ↑ 포토트랜지스터
            빛 통과 → 펄스 발생
```

#### 자기식 엔코더 (Magnetic Encoder)

- 자석이 부착된 디스크 + 홀 센서(Hall Sensor)
- 자기장 변화로 펄스 감지
- 먼지·오염에 강함, 소형 로봇에 적합

### 2.3 채널 구성

| 종류              | 채널 | 특징                                           |
| ----------------- | ---- | ---------------------------------------------- |
| 단채널 (A채널)    | 1개  | 회전량만 측정, 방향 구별 불가                  |
| 2채널 (A+B채널)   | 2개  | 회전량 +**방향** 구별 가능 (90° 위상차) |
| 3채널 (A+B+Z채널) | 3개  | 1회전당 1회 기준 펄스(Index) 추가              |

**2채널 방향 판별 원리:**

```
정방향: A가 B보다 앞섬
A: ▔▔╮___╭▔▔╮___
B: __╭▔▔╮___╭▔▔

역방향: B가 A보다 앞섬
A: __╭▔▔╮___╭▔▔
B: ▔▔╮___╭▔▔╮___
```

## 3. 펄스와 해상도

### 3.1 펄스 (Pulse)

엔코더가 1단계 회전할 때마다 출력하는 전기 신호(HIGH→LOW 또는 LOW→HIGH 전환).

### 3.2 해상도 (PPR: Pulses Per Revolution)

1회전당 발생하는 펄스 수.

| PPR      | 의미                             |
| -------- | -------------------------------- |
| 높을수록 | 더 세밀한 회전 측정 가능         |
| 낮을수록 | 측정 정밀도 낮음, 처리 부담 적음 |

**체배 방식으로 해상도 향상:**

- 1체배(×1): 한 채널 상승 엣지만 카운트
- 2체배(×2): 한 채널 상승+하강 엣지 카운트
- 4체배(×4): A·B 두 채널 상승+하강 모두 카운트 → PPR × 4 효과

예) PPR=30, 4체배 사용 시 → 실효 해상도 = 120 카운트/회전

## 4. 속도 및 거리 계산

### 4.1 기본 공식

```
바퀴 둘레 = π × 바퀴 지름
1펄스당 이동 거리 = 바퀴 둘레 / PPR
```

### 4.2 거리 계산

```
이동 거리 = 총 펄스 수 × (바퀴 둘레 / PPR)
```

**예시:** 바퀴 지름 10cm, PPR=30, 측정 펄스=90

```
바퀴 둘레 = π × 10 = 31.42cm
1펄스당 거리 = 31.42 / 30 = 1.047cm
이동 거리 = 90 × 1.047 = 94.25cm
```

### 4.3 속도 계산

```
속도(cm/s) = (단위시간 내 펄스 수 × 1펄스당 거리) / 측정 시간(s)
```

**예시:** 0.1초 동안 5펄스 측정

```
속도 = (5 × 1.047) / 0.1 = 52.36cm/s
```

## 5. 오차 발생 이유와 보정 방법

### 5.1 오차 발생 이유

| 원인           | 설명                                           |
| -------------- | ---------------------------------------------- |
| 바퀴 미끄러짐  | 바퀴가 헛돌면 펄스는 증가하지만 실제 이동 없음 |
| 바퀴 지름 오차 | 공기압·마모로 실제 지름이 설계값과 다름       |
| 노이즈         | 진동·전기 간섭으로 가짜 펄스 발생             |
| 축 흔들림      | 기구적 유격으로 펄스 카운트 오류               |

### 5.2 보정 방법

| 방법                   | 내용                                                    |
| ---------------------- | ------------------------------------------------------- |
| **캘리브레이션** | 실제 1m 주행 후 측정 거리 비교 → 보정 계수 적용        |
| **IMU 융합**     | 가속도/자이로 센서와 데이터 결합하여 정확도 향상        |
| **슬립 감지**    | 두 바퀴 펄스 차이가 비정상적으로 크면 미끄러짐으로 판단 |
| **디바운싱**     | 소프트웨어로 짧은 시간 내 중복 펄스 필터링              |

## 6. 요약

| 항목      | 핵심 내용                               |
| --------- | --------------------------------------- |
| 역할      | 회전량을 펄스로 변환 → 속도·거리 계산 |
| 구성 방식 | 광학식(슬롯+IR), 자기식(자석+홀센서)    |
| PPR       | 높을수록 정밀, 4체배로 해상도 향상 가능 |
| 거리 계산 | 총펄스 × (둘레/PPR)                    |
| 오차 원인 | 미끄러짐, 지름 오차, 노이즈             |
| 보정 방법 | 캘리브레이션, IMU 융합, 슬립 감지       |
