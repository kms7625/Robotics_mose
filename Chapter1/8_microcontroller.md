# 마이크로컨트롤러(Microcontroller) 조사 보고서

---

## 1. 마이크로컨트롤러의 역할

**마이크로컨트롤러(MCU, Microcontroller Unit)** 란 CPU, 메모리(RAM/ROM), 입출력 장치가 하나의 칩에 집적된 소형 컴퓨터이다.

| 역할 | 내용 |
|------|------|
| 센서 데이터 수집 | IR 센서, 엔코더 등 신호 읽기 |
| 제어 신호 출력 | PWM으로 모터 드라이버 제어 |
| 연산 및 판단 | PID 계산, 경로 판단 등 알고리즘 실행 |
| 통신 | 다른 장치와 데이터 교환 |
| 타이밍 제어 | 인터럽트·타이머로 실시간 제어 |

---

## 2. 기본 제어 방식과 인터페이스

### 2.1 디지털 핀 (Digital Pin)

- HIGH(1) / LOW(0) 두 가지 상태만 처리
- **입력:** 버튼, 엔코더 펄스, IR 센서 디지털 출력
- **출력:** LED 제어, 모터 방향 신호(IN1/IN2)

### 2.2 아날로그 핀 (Analog Pin)

- 연속적인 전압값(0~5V) 읽기
- 내부 ADC(Analog-Digital Converter)로 디지털 값으로 변환
- **용도:** IR 센서 아날로그 출력, 배터리 전압 측정

### 2.3 PWM (Pulse Width Modulation)

- 디지털 핀으로 아날로그 신호 모사
- 듀티 사이클(0~100%)로 평균 전압 조절
- **용도:** 모터 속도 제어, LED 밝기 조절

```
듀티 25%:  ▔╮___╭▔╮___╭▔   → 낮은 평균 전압
듀티 75%:  ▔▔▔╮_╭▔▔▔╮_╭▔   → 높은 평균 전압
```

## 3. 마이크로컨트롤러 통신 방식

### 3.1 UART (Universal Asynchronous Receiver Transmitter)

- **비동기** 직렬 통신, 클럭 신호 없이 미리 정한 보드레이트(Baud Rate) 사용
- 연결선: TX(송신), RX(수신), GND
- **용도:** PC와의 시리얼 통신, 블루투스 모듈, GPS

| 항목 | 내용 |
|------|------|
| 연결 방식 | 1:1 (Point-to-Point) |
| 속도 | 9600 ~ 115200 bps |
| 장점 | 단순, 2선 연결 |
| 단점 | 동시에 1개 장치만 연결 |

### 3.2 I2C (Inter-Integrated Circuit)

- **동기** 직렬 통신, SCL(클럭)과 SDA(데이터) 2선 사용
- 1개 마스터 + 여러 슬레이브 구성 가능 (주소로 구별)
- **용도:** OLED 디스플레이, IMU 센서, 온도 센서

| 항목 | 내용 |
|------|------|
| 연결 방식 | 1:N (최대 127개 슬레이브) |
| 속도 | 100kHz ~ 400kHz |
| 장점 | 2선으로 여러 장치 연결 |
| 단점 | 속도 느림, 거리 제한 |

### 3.3 SPI (Serial Peripheral Interface)

- **동기** 직렬 통신, MOSI/MISO/SCK/CS 4선 사용
- I2C보다 빠르지만 연결선이 많음
- **용도:** SD카드, 디스플레이, 고속 센서

| 항목 | 내용 |
|------|------|
| 연결 방식 | 1:N (CS 핀으로 선택) |
| 속도 | 수 MHz ~ 수십 MHz |
| 장점 | 빠른 속도, 전이중 통신 |
| 단점 | 장치마다 CS 핀 필요 |

### 통신 방식 비교

| 방식 | 선 수 | 속도 | 다중 장치 | 주요 용도 |
|------|-------|------|-----------|-----------|
| UART | 2 | 중간 | 불가(1:1) | PC 통신, BT |
| I2C | 2 | 느림 | 가능(주소) | 센서류 |
| SPI | 4+ | 빠름 | 가능(CS) | SD카드, 디스플레이 |

## 4. 실시간 제어와 타이머

### 4.1 실시간 제어의 필요성

라인트레이서는 **일정 주기마다** 센서 읽기 → PID 계산 → 모터 출력을 반복해야 한다.
`delay()` 함수로 대기하면 그 동안 다른 처리를 못하므로 **타이머 인터럽트** 를 사용한다.

### 4.2 타이머 (Timer)

일정 시간마다 자동으로 인터럽트를 발생시키는 하드웨어 장치.

### 4.3 타이머 사용 절차

1. **타이머 초기화** — 사용할 타이머 번호, 주기 설정
2. **인터럽트 서비스 루틴(ISR) 등록** — 타이머 만료 시 실행할 함수 지정
3. **타이머 시작** — 카운트 시작
4. **ISR 실행** — 지정 주기마다 자동 호출
5. **메인 루프** — 타이머와 별개로 다른 작업 수행 가능

```cpp
// Arduino 예시 (100ms 주기 타이머)
void setup() {
  // 타이머 설정 및 ISR 등록
}

void loop() {
  // 타이머와 독립적으로 실행
}

ISR(TIMER1_COMPA_vect) {
  // 100ms마다 자동 실행
  readSensors();
  computePID();
  setMotorSpeed();
}
```

## 5. 아두이노 (Arduino)

### 5.1 마이크로보드의 개념

**마이크로보드(Microboard)** 란 마이크로컨트롤러 칩을 중심으로 전원 회로, 통신 인터페이스, 입출력 핀을 갖춘 **개발용 보드**이다. 칩만 있는 MCU와 달리 별도 회로 없이 바로 사용 가능.

### 5.2 아두이노의 개념

**Arduino**는 오픈소스 마이크로보드 플랫폼으로, 하드웨어(보드)와 소프트웨어(IDE)를 함께 제공한다.

| 항목 | 내용 |
|------|------|
| MCU | ATmega328P (Uno 기준) |
| 동작 전압 | 5V |
| 클럭 | 16MHz |
| 디지털 핀 | 14개 (6개 PWM 가능) |
| 아날로그 핀 | 6개 (10bit ADC) |
| 플래시 메모리 | 32KB |
| SRAM | 2KB |
| 프로그래밍 | C/C++ 기반 Arduino 언어 |

### 5.3 아두이노 인터페이스 및 통신 방식

| 인터페이스 | 핀 | 설명 |
|------------|-----|------|
| 디지털 I/O | D0~D13 | HIGH/LOW 입출력 |
| PWM 출력 | D3,5,6,9,10,11 | `analogWrite()` |
| 아날로그 입력 | A0~A5 | `analogRead()` → 0~1023 |
| UART | D0(RX), D1(TX) | 시리얼 통신 |
| I2C | A4(SDA), A5(SCL) | `Wire` 라이브러리 |
| SPI | D10(SS), D11(MOSI), D12(MISO), D13(SCK) | `SPI` 라이브러리 |
| 인터럽트 | D2, D3 | 외부 인터럽트 핀 |

### 5.4 라인트레이서에서의 아두이노 활용

```
IR 센서 (A0~A2) ──► Arduino ──► 모터 드라이버 (D3,5,6,9)
엔코더 (D2, D3) ──►            ──► 모터 A / 모터 B
                   │
                   ▼
              PID 연산 → PWM 출력
```
